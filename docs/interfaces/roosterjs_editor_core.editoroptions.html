Optional customData

customData: object
  • Defined in roosterjs-editor-core/lib/interfaces/EditorOptions.ts:79

Initial custom data. Use this option to set custom data before any plugin is initialized, so that plugins can access the custom data safely. The value of this map is the value of each custom data. No disposer function to specify here. Because when set custom data via this way, it means the custom data value is created before editor, so editor shouldn't control the lifecycle of these objects, and caller need to manage its lifecycle.
